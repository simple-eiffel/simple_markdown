MD_TABLE_PROCESSOR

Processes GitHub Flavored Markdown tables

Overview

MD_TABLE_PROCESSOR handles GFM table syntax, converting pipe-delimited rows into proper HTML table markup with thead, tbody, th, and td elements.

Table Format

| Header 1 | Header 2 | Header 3 |
|----------|----------|----------|
| Cell 1   | Cell 2   | Cell 3   |
| Cell 4   | Cell 5   | Cell 6   |

Converts to:

<table>
<thead><tr><th>Header 1</th><th>Header 2</th><th>Header 3</th></tr></thead>
<tbody>
<tr><td>Cell 1</td><td>Cell 2</td><td>Cell 3</td></tr>
<tr><td>Cell 4</td><td>Cell 5</td><td>Cell 6</td></tr>
</tbody>
</table>

Query Features

Feature Signature Description
is_table_row (a_line: STRING): BOOLEAN True if line starts and ends with |
is_separator_row (a_line: STRING): BOOLEAN True if line is |---|---| pattern

Conversion Features

Feature Description
convert_table_start Returns <table>
convert_table_end Returns </table>
convert_header_row Converts header row to <thead><tr><th>...</th></tr></thead>
convert_body_row Converts body row to <tr><td>...</td></tr>
convert_tbody_start Returns <tbody>
convert_tbody_end Returns </tbody>

Inline Processing

Cell contents are processed through MD_INLINE_PROCESSOR, so tables can contain:

  • Bold and italic text
  • Links and images
  • Inline code
  • Strikethrough
